## 3. 🤔 回溯算法是什么？

回溯算法（backtracking algorithm）是一种通过穷举来解决问题的方法，它的核心思想是从一个初始状态出发，暴力搜索所有可能的解决方案，当遇到正确的解则将其记录，直到找到解或者尝试了所有可能的选择都无法找到解为止。

它是一种通过“试错”来寻找问题所有可能解的算法策略。你可以把它想象成在一个迷宫里找路：沿着一条路走下去，如果发现是死胡同，就退回到上一个岔路口，换一条路继续尝试，直到找到所有出口。

它本质上是对解空间树进行深度优先搜索（DFS），但在搜索过程中，一旦发现当前路径不可能得到有效解，就会立即“剪枝”，即放弃对该路径的继续探索，并撤销上一步的选择，回到之前的状态，尝试其他可能性。

## 4. 🤔 什么是「尝试与回退」？

之所以称之为回溯算法，是因为该算法在搜索解空间时会采用“尝试”与“回退”的策略。当算法在搜索过程中遇到某个状态无法继续前进或无法得到满足条件的解时，它会撤销上一步的选择，退回到之前的状态，并尝试其他可能的选择。

- 尝试 ≈ 前进 ≈ push
- 回退 ≈ 撤销 ≈ pop

## 5. 🤔 什么是「剪枝」？

回溯算法效率的关键在于“剪枝”（Pruning）。复杂的回溯问题通常包含一个或多个约束条件，剪枝就是在搜索过程中，通过约束条件提前判断并排除那些不可能产生有效解的路径，从而避免大量的无效尝试，以达到显著提升算法效率的目的。

## 6. 🤔 常见的「剪枝类型」都有哪些？

- 可行性剪枝
  - 提前判断当前路径是否可能达到目标
  - 例如，在“组合总和”问题中，如果当前路径的和已经超过了目标值，就可以立即停止探索
- 去重剪枝
  - 对于包含重复元素的问题，通过排序和跳过重复选择来避免生成重复的解

常见的剪枝类型主要可以归纳为以下几类，它们从不同维度对搜索过程进行优化。

### 6.1. 🚫 可行性剪枝 (Feasibility Pruning)

这是最基础、最常用的剪枝策略。

- 核心思想：在探索路径的过程中，提前判断当前状态是否已经违反了问题的基本约束条件。如果违反，则当前路径及其所有后续分支都不可能产生有效解，应立即停止探索。
- 判断依据：问题的硬性规则。
- 典型应用：
  - 组合总和：当前路径的数字之和已经超过目标值。
  - N皇后问题：在当前行放置皇后时，检查该位置是否与已放置的皇后在同一列或同一对角线上。
  - 0-1背包问题：当前已选物品的总重量已经超过背包容量。

### 6.2. ✂️ 去重剪枝 (Duplicate Pruning)

这种剪枝专门用于处理输入数据中包含重复元素的情况，以确保结果的正确性。

- 核心思想：避免因选择顺序不同但元素相同而产生重复的解。通过规定相同元素的使用顺序，从源头上杜绝重复路径的生成。
- 判断依据：输入数据中的重复元素。
- 典型应用：
  - 全排列 II / 子集 II：输入数组为 `[1, 1, 2]` 时，为了避免生成两个相同的 `[1, 1, 2]` 排列，需要规定两个 `1` 必须按其在数组中的先后顺序被选择。

### 6.3. 🏆 最优性剪枝 (Optimality Pruning)

也称为限界剪枝，专用于求解最优解（如最大值、最小值）的问题。

- 核心思想：在搜索过程中，维护一个当前已知的最优解（例如，目前已找到的最短路径长度）。当探索一条新路径时，如果发现这条路径的“潜力”（例如，当前代价 + 剩余部分的最小可能代价）已经无法超越已知最优解，就立即放弃该路径。
- 判断依据：当前路径的代价与全局已知最优解的比较。
- 典型应用：
  - 旅行商问题 (TSP)：在寻找最短回路时，如果当前已走路径的长度已经超过了之前找到的某条完整回路的长度，就可以停止探索这条路径。
  - 0-1背包问题：估算当前路径下，即使把剩余所有物品都装入，总价值也无法超过已知最大价值时，进行剪枝。

### 6.4. 🔄 记忆化剪枝 (Memoization Pruning)

这种剪枝通过“空间换时间”的策略，避免了对相同子问题的重复计算。

- 核心思想：将已经搜索过的状态及其结果缓存起来。当算法再次遇到完全相同的状态时，直接从缓存中获取结果，而无需重新递归探索。
- 判断依据：当前状态是否已经在缓存中出现过。
- 典型应用：
  - 八数码问题：棋盘的状态可以用一个字符串表示。如果搜索过程中再次到达同一个棋盘布局，说明进入了循环或重复搜索，可以直接剪枝。
  - 带有重叠子问题的递归：许多动态规划问题也可以用带记忆化的回溯（即DFS+记忆化）来解决。

### 6.5. 🎯 其他剪枝技巧

除了以上四大类，还有一些更高级或特定场景下的剪枝策略：

- 对称性剪枝 (Symmetry Breaking)：利用问题本身的对称性来减少搜索空间。例如，在图着色问题中，可以强制规定第一个顶点必须使用1号颜色，从而消除因颜色编号互换而产生的大量等价解。
- 启发式剪枝 (Heuristic Pruning)：根据经验或特定规则（启发式函数）来评估一条路径的“好坏”，优先探索更有希望的路径，并提前放弃那些看起来希望不大的分支。

总而言之，一个高效的回溯算法往往是多种剪枝策略的组合应用。理解并灵活运用这些剪枝类型，是掌握回溯算法的关键。

## 7. 🤔 回溯算法的「通用框架代码」是什么样的？

理解回溯算法有一个非常清晰的通用框架。解决一个回溯问题，实际上就是遍历一棵“决策树”的过程。

```python
result = []  # 用于存储所有找到的解

def backtrack(路径, 选择列表):
    # 1. 结束条件：如果当前路径满足目标，说明找到了一个解
    if 满足结束条件:
        result.append(路径.copy())  # 记录当前路径
        return

    # 2. 遍历所有可能的选择
    for 选择 in 选择列表:
        # 3. 剪枝：如果当前选择不满足约束，则跳过
        if not 满足约束条件(选择):
            continue

        # 4. 做选择：将选择加入路径，并更新选择列表
        路径.append(选择)
        从选择列表中移除(选择)

        # 5. 探索：基于当前选择，递归地探索下一步
        backtrack(路径, 选择列表)

        # 6. 撤销选择（回溯）：探索完毕后，撤销上一步的选择，恢复状态
        路径.pop()
        将选择加回选择列表
```

这个框架清晰地展示了回溯算法“做选择 -> 递归探索 -> 撤销选择”的核心流程。

## 8. 🤔 回溯算法有哪些「常用术语」？

核心思想：选择、探索与撤销

回溯算法的核心可以概括为一个“前进-失败-后退-再前进”的循环过程。这个过程由三个关键要素驱动：

1. 选择 (Choice)：在每一步，你都有一个可选的“选择列表”。你需要从中做出一个选择，从而向解前进一步。
2. 约束 (Constraint)：做出选择后，需要判断当前路径是否满足问题的约束条件。如果不满足，就进行“剪枝”，不再继续探索这条路径。
3. 目标 (Goal)：当路径满足结束条件时，说明你已经找到了一个有效的解，可以将其记录下来。

## 10. 🤔 回溯算法的「典型例题」有哪些？

回溯算法虽然时间复杂度通常很高（如 O(n!) 或 O(2^n)），但对于许多需要枚举所有可能解的问题，它是最直观有效的解决方法。常见应用包括：

- 组合问题：从 n 个元素中选出 k 个元素的所有组合。
- 排列问题：生成一个集合中所有元素的全排列。
- 子集问题：生成一个集合的所有子集。
- 棋盘问题：经典的 N 皇后问题、数独求解。
- 路径搜索问题：在迷宫或网格中寻找所有可行路径。
